## Security Considerations

### Blocked Command Patterns
```python
BLOCKED_PATTERNS = [
    "rm -rf /",
    "rm -rf /*",
    "mkfs",
    "dd if=/dev/zero of=/dev",
    ":(){ :|:& };:",
    "> /dev/sda",
    "chmod -R 777 /",
]
```

### Authentication Layers
1. **SSL/TLS** - All traffic encrypted via Caddy
2. **API Keys** - Stored in environment variables
3. **IP Filtering** - Optional whitelist for sensitive endpoints
4. **Rate Limiting** - Prevents abuse

### Secrets Management
- No hardcoded credentials
- Environment variables via systemd
- `.env` files excluded from git
- Telegram alerts for suspicious activity

## Deployment Process

### Adding a New Service
```bash
# 1. Create Python file
vim /home/ubuntu/grok-voice/grok-newservice.py

# 2. Create systemd service
sudo vim /etc/systemd/system/grok-newservice.service

# 3. Enable and start
sudo systemctl daemon-reload
sudo systemctl enable grok-newservice
sudo systemctl start grok-newservice

# 4. Add to Caddy (if needed)
sudo vim /etc/caddy/Caddyfile
sudo systemctl reload caddy
```

### Or use Agent Mode
```python
agent.create_service(
    name="newservice",
    code=python_code,
    port=8080,
    description="My New Service"
)
```
